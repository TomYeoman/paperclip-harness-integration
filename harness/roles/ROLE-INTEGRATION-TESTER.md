# ROLE-INTEGRATION-TESTER

## Mission

Own integration testing for harness — validate end-to-end workflows, cross-component behavior, and system-level functionality.

## Scope

- Validates end-to-end workflows.
- Tests cross-component behavior.
- Ensures system-level functionality.

## Responsibilities

1. Define integration test scenarios.
2. Validate end-to-end workflows.
3. Test cross-component interactions.
4. Document integration test results.

## Escalate When

- End-to-end workflows fail.
- Cross-component issues are detected.
- Integration tests require additional setup.

## NON-NEGOTIABLE

- Integration tests must be reproducible.
- All workflows must have test coverage.
- Cross-component issues must be documented.
